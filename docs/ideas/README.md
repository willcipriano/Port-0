# Port 0 — Expansion Ideas

> Status: Brainstorm | Not implementation scope

This folder holds **post-MVP design explorations**. Nothing here is locked for build unless it graduates into [`../../spec/`](../../spec/) with an explicit decision and phase target.

Ideas should stay compatible with MVP pillars: server-authoritative sim, hybrid real-time + tick time, hidden ownership, and fleet-vs-fleet conflict — but may propose new player paths, economies, or systems.

## How to use these docs

1. Treat each file as a **design pitch** with open questions, not a task list.
2. When an idea is approved, extract decisions into the relevant spec doc and add a row to [15-mvp-scope.md](../../spec/15-mvp-scope.md) phased roadmap.
3. Implementation plans belong in [`../../plan/`](../../plan/) only after spec acceptance.
